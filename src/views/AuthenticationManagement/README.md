# 认证管理界面

## 功能概述

这是一个现代化的认证管理界面，用于管理用户的身份认证申请。界面采用卡片式设计，提供直观的数据展示和高效的审核流程。

## 主要功能

### 1. 统计概览

- **待审核**: 显示待处理的认证申请数量
- **已通过**: 显示已通过的认证申请数量
- **已拒绝**: 显示已拒绝的认证申请数量
- **总申请数**: 显示所有认证申请的总数

### 2. 筛选功能

- **状态筛选**: 可按全部/待审核/已通过/已拒绝状态筛选
- **身份筛选**: 可按学生/教师/校友身份筛选
- **刷新按钮**: 重置筛选条件并刷新数据

### 3. 认证申请卡片

每个认证申请以卡片形式展示，包含：

- 用户基本信息（头像、姓名、身份标签、手机号）
- 申请状态标签
- 详细信息（性别、用户ID、审核时间、拒绝原因等）
- 认证材料图片（支持预览）
- 操作按钮（通过/拒绝/详情）

### 4. 审核操作

- **通过认证**: 一键通过用户的认证申请
- **拒绝认证**: 填写拒绝原因后拒绝申请
- **查看详情**: 查看完整的认证申请信息

### 5. 分页功能

- 支持自定义每页显示数量（12/24/36/48）
- 页码跳转
- 显示总数统计

## 设计特点

### 现代化卡片布局

- 采用响应式网格布局，自动适配不同屏幕尺寸
- 卡片悬停效果，提升交互体验
- 使用渐变背景和阴影增强视觉层次

### 色彩系统

- **待审核**: 黄色系（#f39c12）
- **已通过**: 绿色系（#27ae60）
- **已拒绝**: 红色系（#e74c3c）
- **总数**: 蓝色系（#3498db）

### 动画效果

- 卡片进入/离开动画
- 悬停缩放动画
- 平滑过渡效果

### 暗黑模式支持

- 完整支持暗黑模式
- 自动适配颜色和对比度

## 技术实现

### API 接口

使用 `身份认证控制类` 的以下接口：

- `postAuthIdentityRequestsPage`: 分页查询认证申请
- `postAuthIdentityRequestsHandle`: 处理认证申请（通过/拒绝）

### 数据字典

- `DICT_TYPE.GENDER`: 性别字典
- `DICT_TYPE.USER_TYPE`: 用户身份字典

### 响应式设计

- 桌面端：4列网格布局
- 平板端：2-3列网格布局
- 移动端：单列布局

需要 `auth:page` 权限才能访问此页面。
