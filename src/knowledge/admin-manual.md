# 深大校友录管理后台 - 知识使用手册

---

## 系统概述

深大校友录管理后台是深圳大学校友管理平台的 Web 端管理系统。系统采用 RBAC（基于角色的访问控制）权限模型，管理员通过分配的角色和权限访问不同功能模块。系统支持暗色模式，侧边栏可折叠，所有页面均通过路由权限守卫进行访问控制。

系统地址访问根路径 `/` 会自动跳转到登录页 `/login`，登录成功后跳转到首页 `/home`。

---

## 登录与注册

### 登录

- 访问路径：`/login`
- 无需登录即可访问
- 输入用户名和密码进行登录
- 支持「记住账号」功能，勾选后下次自动填充用户名
- 登录成功后跳转至首页，若从其他页面被重定向到登录页，登录后自动返回原页面
- 已登录用户访问登录页会自动跳转到首页

### 注册

- 访问路径：`/register`
- 无需登录即可访问
- 填写用户名、昵称、密码、确认密码
- 需勾选服务协议方可注册
- 注册成功后跳转至登录页

---

## 首页（仪表盘）

- 访问路径：`/home`
- 所需权限：仅需登录
- 展示当前登录用户的基本信息：头像、用户名、昵称、用户 ID
- 展示 Token 信息（用于开发调试）
- 展示当前用户拥有的权限列表和权限树结构
- 提供快捷操作按钮：刷新数据、编辑资料、系统设置、退出登录
- 提供调试用 JSON 查看器

## 帖子批量导入

- 访问路径：`/post-batch-import`
- 所需权限：`post:page`（页面访问）
- 功能说明：通过 AI 辅助将原始文本批量改写并发布为帖子

### 四步流程

#### 第一步：输入原始文本

- 支持添加多条文本输入
- 每条文本作为一个独立的帖子原料

#### 第二步：AI 改写

- 系统调用 Coze 工作流对每条文本进行 AI 改写
- 显示并发改写进度
- 改写失败的条目支持重试
- 可从原始 JSON 手动解析失败条目

#### 第三步：预览与编辑

- 预览 AI 改写后的帖子内容
- 支持手动修改调整

#### 第四步：批量发布

- 通过机器人批量发帖接口一键发布所有帖子

## 权限系统说明

### RBAC 模型

系统采用基于角色的访问控制（RBAC）：

- 权限（Permission）以树形结构组织
- 角色（Role）通过勾选权限树节点获得一组权限
- 用户（User）通过被分配角色获得对应权限
- 超级管理员（isAdmin）自动拥有所有权限，无需单独分配

### 路由权限守卫

- 每个路由在 meta 中定义 `permission` 字段指定所需权限码
- 未登录用户访问需认证页面会被重定向到登录页
- 已登录但无权限的用户会被重定向到首页
- 标记 `adminOnly` 的路由仅超级管理员可访问

### 全部页面权限码对照表

| 页面         | 路径                         | 权限码               |
| ------------ | ---------------------------- | -------------------- |
| 首页         | `/home`                      | 无（仅需登录）       |
| 学院管理     | `/college-management`        | `college:page`       |
| 班级管理     | `/class-management`          | `class`              |
| 校友档案管理 | `/alumni-management`         | `alumni:page`        |
| 用户管理     | `/user-management`           | `user:page`          |
| 管理员管理   | `/admin-management`          | `admin-user:page`    |
| 角色管理     | `/role-management`           | `role:page`          |
| 权限管理     | `/permission-management`     | `permission:page`    |
| 字典管理     | `/dict-management`           | `dict:page`          |
| 操作日志统计 | `/operation-log-statistics`  | `operation-log:page` |
| 认证管理     | `/authentication-management` | `auth:page`          |

## 认证与会话

### 登录认证

- 系统使用 JWT Token 进行身份认证
- 登录成功后 Token 存储在本地，后续请求自动携带
- Token 失效时用户会被重定向到登录页

## 常见问题

### 无法访问某个页面

检查当前账号的角色是否拥有该页面对应的权限码。联系超级管理员在「角色管理」中为你的角色分配对应权限。

### 如何为新管理员开通账号

1. 在「管理员管理」中创建管理员账号
2. 在「角色管理」中确认或创建合适的角色
3. 为该角色在「角色管理 > 分配权限」中勾选所需权限
4. 在「用户管理」或「管理员管理」中为新管理员分配角色

### 字典数据修改后页面未更新

字典数据修改后系统会自动清除缓存。如果仍未更新，尝试刷新页面。

### 帖子批量导入失败

1. 检查输入的原始文本是否为空
2. AI 改写失败的条目可点击重试
3. 如果 Coze 工作流返回原始 JSON，可使用「手动解析」功能提取结果
4. 确认网络连接正常

### 推荐系统初始化注意事项

初始化推荐系统会调用 DashScope API 产生费用，请确认已获得授权后再操作。初始化包含四个步骤，需要等待全部完成。

### 认证审核驳回原因要求

驳回身份认证时，驳回原因至少需要输入 5 个字，请提供具体的驳回理由。
